# docs/

Design system documentation, decision records, and onboarding material.

## Suggested layout (add as needed)

```
docs/
├── design-system/      # Tokens, components, usage guidelines
├── brand/              # Voice, logo rules, photography direction
├── decisions/          # Architecture Decision Records (ADRs)
└── onboarding/         # How to run things locally, contribute, etc.
```

Keep docs close to the code they describe. A README beside a `packages/ui/` component is often better than a doc here — use this folder for cross-cutting material.
